- 2026-06-08 TIME-ADVANCE APPROACH chosen = approach (a), global clock. PassTrancheN calls Convex.Class.setPOSIXTime <trancheNDate> (jumps mockchain "now" to that vest date) and flips the model flag. The validator's IRV check uses contains (from vDate) (txInfoValidRange) — inclusive lower bound — so once the clock >= vDate, a Withdraw tx whose validity-range LOWER BOUND = current "now" will be contained in [vDate, +inf) and count that tranche as vested. Rationale: setPOSIXTime exists in Convex.Class and is available in TestingMonadT (MonadFail+MonadMockchain); no example drives POSIXTime validity but the API is clear. Monotonicity: PassTranche2 sets BOTH modelT1Passed and modelT2Passed (reaching the later date implies the earlier is past) so model flags never disagree with the monotonic clock. OPEN for Withdraw step: PassTranche1 emitted AFTER PassTranche2 would request a backwards clock move (tranche1Date < tranche2Date); Withdraw impl must derive its validity range from actual chain "now" / the model flags and account for whether setPOSIXTime permits moving backwards (treat flags as the source of truth for what's vested, set the Withdraw validity lower bound to max passed vDate).
